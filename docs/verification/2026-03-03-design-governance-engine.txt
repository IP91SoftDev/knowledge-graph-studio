=== PHASE 1 — HARD CONSTRAINTS SYSTEM ===

**File Created:** docs/design/hard-constraints.md (8.4 KB)

**Constraints Defined:**

### Constraint 1: Depth Layer Rules
- 4-level elevation system ONLY (Level 0-3)
- Level 0: Page background (z-0)
- Level 1: Section surface (z-10)
- Level 2: Card surface (z-20)
- Level 3: Hover/Active (z-30)
- No Level 4+ allowed

### Constraint 2: Spacing Tier Enforcement
- Strict scale: 4/8/16/24/32/48/64px
- Section-to-section: xl (32px) or 2xl (48px)
- Card padding: md (16px) or lg (24px)
- Button internal: sm (8px) or md (16px)
- No arbitrary values allowed

### Constraint 3: Accent Saturation Limits
- Max 1 primary accent per project
- Accent ≤ 10% of viewport
- No rainbow UI
- No emoji in SaaS mode

### Constraint 4: Motion Duration System
- Fast: 120ms (hover)
- Normal: 180ms (expand/collapse)
- Smooth: 240ms (page transitions)
- No animation > 300ms
- No spring chaos

**Enforcement Rule:** If constraint violated → REJECT
